# 评论区运营与舆情应对

## 核心姿态

评论区不是笔记的附属品，**它是内容的第二现场**。

用户看完正文，会滑到评论区做"二次验证"。**评论区的第一屏，往往比正文更能决定他信不信你。**

但这里有一条清晰的红线：**预案、真实互动、正当置顶 = 正常运营；刷评、控评、雇水军 = 违规。** 这个 skill 只做前者。

## 发布前：写预案，不是临场发挥

- 预判这条内容会引发的 **3-5 类典型评论**（好奇型、质疑型、经验分享型、抬杠型、负面型）
- 为每一类准备**回应方向**（不是逐字话术，避免机械感）
- 提前想好 **3 个你希望用户问的问题**，可以由品牌自己在评论区带头问出来（这是正常的话题引导，不是伪装成用户）
- 明确**红线问题清单**：哪些问题不能在评论区回答，要走客服或公关

## 发布后：时效是关键

| 时间窗 | 动作 |
|---|---|
| **0-24 小时** | 最关键。逐条看评论，真诚回复，置顶最有价值的一条 |
| 24-72 小时 | 持续维护，补充信息，处理长尾疑问 |
| 72 小时后 | 定期回访；有价值的评论可持续置顶 |

**头 24 小时别撒手。** 这个窗口既影响用户观感，也影响内容的互动数据。

## 置顶策略

置顶的那一条，等于你替读者划的重点。可以用来：

- **补充正文没说完的信息**（最实用）
- 澄清一个高频误解
- 放大一条真实的优质用户反馈
- 抛出一个引导讨论的问题

**不要置顶纯吹捧的评论**，读者一眼就能看出来，反而降低可信度。

## 负面分级与响应

| 级别 | 特征 | 响应 |
|---|---|---|
| L1 吐槽 | 个人体验不佳、情绪化但无实质指控 | 真诚回应，不辩解，能解决就给解决路径 |
| L2 质疑 | 对产品、价格、宣传提出具体质疑 | 用事实回应，不删评，不硬刚 |
| L3 事实性问题 | 指出真实的产品或服务缺陷 | **承认 + 给出处理动作**，向内部反馈 |
| L4 危机 | 涉及安全、法律、歧视、群体性情绪 | **立即升级**，不在评论区自行回应，走公关流程 |

**通用原则**：
- **删评论会激化事态**，除非涉及违法、辱骂、隐私泄露或垃圾广告。
- 承认问题的品牌，通常比坚决否认的品牌恢复得快。
- 负面是**产品和服务的免费反馈**，要有回流到业务侧的机制。

## 升级路径（必须提前定）

在没有压力的时候，先写清楚：
- 谁有权在评论区代表品牌发言
- 什么级别的问题必须升级，升级给谁
- 多久内必须给出第一次回应
- 什么情况下需要法务/公关介入

**热搜发生的时候再讨论流程，就已经晚了。**

## 风险边界

- **不提供任何刷评、控评、雇水军、批量小号或伪造用户身份的做法。** 这些行为违反平台规则，也可能构成虚假宣传。
- 品牌在评论区发言应可被识别为品牌方，不得伪装成普通用户做推荐。
- 不得删除或压制合法的批评意见（违法、辱骂、隐私、垃圾广告除外）。
- 不得在评论区收集用户个人信息或引导到不合规的私域路径。
- 危机事件的处理涉及法律风险，重大事项应由法务与公关负责，本 skill 不替代专业意见。
